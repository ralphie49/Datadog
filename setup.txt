2) HOW TO RUN IT
-----------------------------------------------------------------
PROMPT:

Run the **Datadog Analyser Orchestrator**: read `datadog-analyser-orchestrator.md`,
then follow each of the 10 agent `.md` files it references, in order, phase by
phase — obeying every MUST/MUST NOT rule literally. Use your export files from
an input folder of your choosing, write each phase's output to `output/` with the
exact filenames specified, and finish by generating `output/datadog_analysis_report.md`.
If code generation is used, generated `run_datadog_analysis.py` should be placed
inside the matching `output/<dataset>/` subfolder for that dataset.
-----------------------------------------------------------------


3) WHAT SHOULD COME OUT
--------------------------
After a full run, output/ should contain one subfolder per input dataset,
and each subfolder should contain these 11 files:

  normalised_data.json       (Phase 0 — ingestion)
  log_analysis.json          (Phase 1 — errors + data quality)
  metrics_report.json        (Phase 2 — latency/throughput/host health)
  apm_report.json            (Phase 3 — Kafka lag/checkpoints/SLA)
  security_report.json       (Phase 4 — PII/credential/auth findings)
  anomaly_report.json        (Phase 5 — spikes/drops/trends)
  dependency_report.json     (Phase 6 — service graph + breakpoints)
  root_cause.json            (Phase 7a — correlated incidents)
  recommendations.json       (Phase 7b — ranked fixes)
  patch_suggestions.json     (Phase 8 — draft diffs, human review only)
  datadog_analysis_report.md (Phase 9 — final consolidated report)

If any of these are missing from a dataset subfolder, check the assistant's run
log for which phase stopped and why — the agent specs are written to fail
loudly (stop + report) rather than silently skip a phase.
